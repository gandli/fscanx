## BL-001 添加 -dp 命令行选项

- 模块：命令行参数
- 优先级：P1
- 状态：已完成
- 当前阶段：持续迭代
- 价值：用户对纯域名目标扫描时，可以自动组合 -p 指定的所有端口，无需手动逐个构造 URL。
- 关联模块文档：
- 关联计划：plans/2026-06-11-BL-001-dp-flag.md
- 关联代码：common/flag.go, common/config.go, common/ParseIP.go
- 验收状态：未验收

### 范围

- 新增 `-dp` 命令行选项（bool 类型）
- 当 `-dp` 启用时，对于纯域名输入（如 `baidu.com`），自动用 `-p` 指定的端口生成 `http://域名:端口` 和 `https://域名:端口` 格式的 URL
- 例如：输入 `baidu.com`，`-p 443,80` → 生成 `http://baidu.com:80`, `http://baidu.com:443`, `https://baidu.com:80`, `https://baidu.com:443`

### 不做

- 不改变非域名目标的处理逻辑
- 不改变未启用 `-dp` 时的行为
- 不处理已经是完整 URL 的输入（`-u` 参数）

### 验收条件

- [ ] `-dp` 标志能正常解析
- [ ] `-hf` 文件中的纯域名行，启用 `-dp` 后按 `-p` 端口生成所有 http/https URL 组合
- [ ] `-h` 参数中的纯域名，启用 `-dp` 后按 `-p` 端口生成所有 http/https URL 组合
- [ ] 未启用 `-dp` 时行为不变（纯域名默认生成 `http://域名` 和 `https://域名`）
